# Cryptographic Image Signing & Attestation

This directory outlines the mechanism by which we verify the integrity of the Gateway and Worker artifacts, defending against unauthorized modification.

## Purpose
An adversary could theoretically inject malicious code into the container running our policy engine. Image Signing guarantees non-repudiation: we cryptographically sign the container after building it so that the runtime environment (or an auditor) can mathematically prove the container was authored by us.

Attestations take this a step further: we bind the SBOM generated by `syft` to the image signature using the **in-toto** specification. This proves not only who built the container, but what dependencies exactly were inside it at the exact moment it was signed.

## Implementation Details
We use [Cosign](https://github.com/sigstore/cosign), part of the Sigstore project, to orchestrate signing and verification.

For our local environment, we manage this using explicit keypairs:
1. `cosign generate-key-pair` creates a `.keys/cosign.key` and `.keys/cosign.pub`.
2. `./scripts/sign_images.sh` pushes the local dev images to a local ephemeral registry, signs them, and attaches the SBOM.
3. `./scripts/verify_provenance.sh` validates the signature and unbundles the attestation.

To see how this local strategy scales to GitLab OIDC + Keyless signing in production environments, reference the `supplychain/build-pipeline-strategy.md`.
